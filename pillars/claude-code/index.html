    Claude Code 完全ガイド¶

    公式ドキュメントを読む前に、実際に使い込んで分かったことを先に押さえよう。
    タブでレベルを選び、ステップ順に進めれば迷いません。

    使ってみて分かったこと

    Claude Codeは「プロンプトを書くツール」ではなく「設計を伝えるツール」。最初にHooksと権限設計を押さえるかどうかで、生産性が桁違いに変わります。

    まずはインストールから基本操作まで。1日で動かせるようになります。

    Step 1 — 環境を整える（5〜15分）

    • インストールガイド — Windows PowerShell 1行コマンド対応。Mac/Linuxも網羅
    • Web版入門ガイド — ローカル環境なしで試したい人はこちら

    Step 2 — 基本操作を覚える（30分〜1時間）

    • Claude Code 完全ガイド — 機能・設定・運用を網羅。リファレンスとして手元に
    • 活用法・基本編 — 日常開発で使えるベストプラクティス。ここが生産性に直結
    • タスク管理 BP — プロジェクト管理のパターン

    📝 初心者へのアドバイス: 完全ガイドは長いので、最初はインストールガイド → 活用法・基本編の順で十分。完全ガイドはリファレンスとして使うのがおすすめ。

    Hooks・権限設計・CI連携で「手放し運転」に近づける。

    Step 3 — Hooks/自動化を組む（1〜2時間）

    • Hooks 完全ガイド — 自動化の起点。基礎から応用まで
    • Hooks 高度な制御 — 条件付き実行とワークフロー制御。本番運用で必須
    • 自動化の全体像 — 自動化パターンの俯瞰マップ
    • Cron 自動化ガイド — 定期実行でルーティンを効率化

    Step 4 — 権限とCI/CDを固める

    • 権限管理概要 — 安全に自律動作させるための設計
    • 自動承認設定ガイド — 手動確認を最小化する設計
    • Sandbox 安全自動承認 — 危険オプションなしで実現する安全設定
    • GitHub Actions 連携 — CI/CDパイプラインへの統合
    • スケジュール実行ガイド — GitHub Actions Scheduledで自動化

    💡 実践者のポイント: Hooksを最初にセットアップすると、権限設計の必要性が自然と見えてきます。「Hooks → 権限 → CI連携」の順がスムーズ。

    サブエージェント・MCP・SDK活用でClaude Codeの能力を最大限引き出す。

    Step 5 — MCP/サブエージェントで拡張（2〜3時間）

    • サブエージェント入門 — 並列処理で分析品質が飛躍的に向上
    • サブエージェント実践 — ハンズオンで学ぶ実装方法
    • MCP 統合戦略 — 拡張性を極めるMCP連携設計
    • MCP 実装ガイド — 外部データ連携からリアルタイム処理まで
