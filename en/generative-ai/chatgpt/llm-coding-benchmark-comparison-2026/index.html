      LLM Coding Benchmark Comparison (April 2026)¶

      How to Read GPT-5.4, Claude, and Gemini 3.1¶

      What you'll learn

      Which benchmark assumptions from February and March 2026 are already stale Where GPT-5.4, Claude Opus 4.6 / Sonnet 4.6, and Gemini 3.1 Pro currently sit How to read SWE-Bench Pro, Terminal-Bench 2.0, OSWorld, pricing, and long context in the current market

      Several benchmark articles written just a few weeks ago are already drifting out of date. The biggest change is simple: OpenAI's center of gravity has moved from GPT-5.3-Codex to GPT-5.4, and GPT-5.4 is now rolling across ChatGPT, the API, and Codex. At the same time, Anthropic has clarified a two-layer structure of Opus 4.6 as the flagship and Sonnet 4.6 as the practical default, while Google continues to position Gemini 3.1 Pro as its most advanced model for complex tasks.

      That means the job of a current benchmark article is no longer to preserve an old leaderboard snapshot. It is to explain which benchmark axes still matter for product choice right now.

      Short answer¶

      If you're asking...Current first answer
      Which model has the strongest public signal for CLI autonomy and coding loops?GPT-5.4 / Codex
      Does Claude still matter at the top end?Yes. Opus 4.6 is still a serious flagship, but Sonnet 4.6 now matters much more operationally
      Is Gemini 3.1 Pro still relevant?Yes, especially for Google's stack, multimodal work, long context, and pricing efficiency
      Is a GPT-5.3-Codex-era title still good enough?No. April 2026 comparisons should move to GPT-5.4

      The baseline changed¶

      OpenAI¶

      OpenAI released GPT-5.4 on March 5, 2026 and explicitly rolled it out across ChatGPT, the API, and Codex. OpenAI also describes GPT-5.4 as its first mainline reasoning model to incorporate the frontier coding capabilities of GPT-5.3-Codex, while expanding further into professional work and computer use.

      So the OpenAI side of any current benchmark comparison should now be anchored on GPT-5.4, not GPT-5.3-Codex.

      Anthropic¶

      Anthropic positions Claude Opus 4.6 as its most capable model and emphasizes professional software engineering, complex agentic workflows, and high-stakes enterprise tasks. At the same time, Claude Sonnet 4.6 is now the default model in claude.ai and Claude Cowork, with pricing that stays at $3 / $15 per million tokens.

      That means a realistic Claude-side comparison has two levels:

      • Opus 4.6 for the flagship ceiling
      • Sonnet 4.6 for the operational default

      Google¶

      Google DeepMind's model card still describes Gemini 3.1 Pro as Google's most advanced model for complex tasks, with 1M context and 64K output. Gemini pricing remains aggressive, starting at $2 input / 12 output** for prompts up to 200K tokens and rising to **4 / $18 above that threshold.

      But Google's own comparison table was published in February 2026, before GPT-5.4 shipped. So Gemini 3.1 Pro remains relevant, while its older published head-to-head framing is no longer the full current picture.

      The five benchmark axes that matter now¶

      1. SWE-Bench Pro¶

      This is still one of the best public anchors for repository issue resolution. OpenAI now publishes GPT-5.4 at 57.7% on SWE-Bench Pro (Public), above GPT-5.3-Codex at 56.8% and GPT-5.2 at 55.6%.

      The important shift is not just the number. It is that OpenAI has clearly moved the benchmark conversation forward from GPT-5.3-Codex to GPT-5.4.

      2. Terminal-Bench 2.0¶

      This remains the cleanest public signal for CLI-style autonomy. OpenAI publishes GPT-5.4 at 75.1%. Anthropic's Opus 4.6 page publishes 65.4%.

      At the moment, this is the most straightforward benchmark axis on which to say: the public official signal currently favors GPT-5.4 for CLI-heavy coding workflows.

      3. OSWorld¶

      This is where older articles are now most likely to be wrong. A common older simplification was: "Codex for terminal work, Claude for GUI work." That no longer holds cleanly. OpenAI publishes GPT-5.4 at 75.0% on OSWorld-Verified, while Anthropic publishes 72.7% for Opus 4.6.

      So if you are still carrying the old assumption that browser or desktop automation automatically belongs to Claude, you should update that assumption.

      4. Pricing¶

      At a high level, the current published prices look like this:

      ModelInputOutputNotes
      GPT-5.4$2.50$15Main OpenAI comparison anchor for Codex/API
      Claude Opus 4.6$5$251M context is beta on Claude Platform
      Claude Sonnet 4.6$3$15Default in claude.ai and Claude Cowork
      Gemini 3.1 Pro$2 or $4$12 or $18Price tier changes above 200K tokens

      At the flagship tier, GPT-5.4 is cheaper than Opus 4.6. On the Claude side, Sonnet 4.6 is the price-performance model that now matters in almost every serious operational comparison. On the Google side, Gemini 3.1 Pro remains price-competitive, especially for shorter prompts.

      5. Long context¶

      OpenAI's GPT-5.4 has a 1.05M context window, though requests beyond 272K input tokens carry higher pricing multipliers. Anthropic highlights a 1M token context window in beta for Opus 4.6 and Sonnet 4.6. Gemini 3.1 Pro also supports 1M context.

      By April 2026, sheer context length is no longer the decisive differentiator. The more useful question is: what kind of work do you want to run inside that context?

      • CLI coding loops and autonomous repo work: GPT-5.4
      • Deep multi-step enterprise or agentic work: Opus 4.6
      • Google-native multimodal and long-context work: Gemini 3.1 Pro

      What should be updated in benchmark articles now?¶

      1. Titles¶

      Older titles are already losing relevance.

      • Titles centered on GPT-5.3-Codex are stale
      • Broad comparison titles should move to GPT-5.4 / Claude / Gemini 3.1
      • Codex-vs-Claude articles should foreground GPT-5.4 vs Opus 4.6

      2. Conclusions¶

      The most important outdated conclusion to retire is this:

      • do not keep repeating "Codex for terminal, Claude for GUI" as a fixed rule
      • do not treat Claude as a single-model story
      • do not treat Google's February comparison sheet as the current full leaderboard

      3. Internal journey¶

      The most natural reading flow now is:

      1. broad comparison
      2. Codex vs Claude
      3. Gemini's updated position
      4. practical onboarding guides

      Summary¶

      • As of April 2026, the broad comparison should be read across GPT-5.4, Opus 4.6, Sonnet 4.6, and Gemini 3.1 Pro
      • OpenAI's side has shifted decisively to GPT-5.4
      • Anthropic should be read through Opus 4.6 as flagship and Sonnet 4.6 as default
      • Gemini 3.1 Pro remains relevant, but Google's published comparison framing is still pre-GPT-5.4
      • A current benchmark article should optimize less for preserving old rankings and more for showing which comparison axis is useful right now
