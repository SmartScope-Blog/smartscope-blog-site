    Codex CLI Update Not Reflecting? 4 Proven Fixes for v0.99+ (Cache, Duplicates, Fresh Install)¶

    Codex CLI Complete Guide

    What you’ll solve in this guide

    Identify why Codex CLI refuses to pick up the latest release
    Apply four fixes: version sanity check, shell cache reset, duplicate uninstall, clean reinstall
    Implement preventative measures so the issue never returns

    If you've run npm install -g @openai/codex@latest or brew upgrade codex yet codex --version still reports an older build, you are not alone. Codex CLI updates routinely fail when duplicate installers, cached paths, shell command caches, or the version-reporting bug get in the way. Apply these four fixes with cache clearing and reinstall steps—version sanity check, cache reset, duplicate uninstall, clean reinstall—to guarantee the newest build launches.

    Target Audience

    • Developers who update Codex CLI via npm or Homebrew but still launch an old binary
    • Tech leads who need to eliminate version drift across a Codex CLI rollout

    Key Points¶

    1. Pinpoint the exact Codex binary your shell is invoking
    2. Remove duplicate installs and stale cache entries
    3. Confirm that the newest Codex CLI actually launches

    Why Codex CLI updates fail to apply¶

    Three failure modes explain almost every “update didn’t stick” report.

    The big three root causes¶

    Installing Codex via npm, Homebrew, winget, or manual binaries leaves multiple executables behind. The oldest often appears first in PATH, especially on Windows where %AppData%\npm, C:\Program Files\Git\usr\bin, and ad-hoc copies coexist.

    Bash and Zsh cache command locations. Unless you open a new terminal (or run hash -r / rehash) after upgrading, the shell keeps using the stale path.

    As documented in GitHub Issue #4986, codex --version can report the wrong release. The binary is current, but the output string lags behind.

    Heads-up for Windows users

    The official docs still list native Windows support as experimental. If stability matters, install Codex CLI inside WSL and run it from there.

    3-minute diagnostic for “codex cli update command” issues¶

    Fast checklist when codex cli update refuses to stick
    1. Version sanity check — compare codex --version with npx -y @openai/codex@latest --version to rule out the version-reporting bug.
    2. Hunt for duplicates — run which -a codex (macOS/Linux) or Get-Command codex -All (PowerShell) to see if multiple executables exist. If yes, uninstall every copy before proceeding.
    3. Reinstall + PATH verify — standardize on a single installer (npm install -g @openai/codex@latest or brew install codex), open a brand-new terminal, and confirm only one path remains via which codex / Get-Command.

    This quick workflow answers the most common “codex cli update command” search intent: why didn’t the new version show up and how do I guarantee it does?

    Three diagnostic command sets¶

    Start by discovering which executable the shell calls and how npm currently tracks Codex.

    # Windows PowerShell
Get-Command codex -All | Select-Object -Expand Source
where.exe codex
npm root -g
npm ls -g @openai/codex --depth=0

    # macOS / Linux
which -a codex
npm root -g
npm ls -g @openai/codex --depth=0

    Seeing multiple paths from Get-Command codex -All or which -a codex confirms duplicate installations. When /usr/local/bin/codex and ~/.npm-global/bin/codex coexist, shell precedence dictates which one launches.

    Bypass PATH with npx

    npx -y @openai/codex@latest --version runs the latest package straight from npm. If that prints a newer version than codex --version, you’ve proven it’s a PATH problem.

    Three steps to guarantee an update¶

    With the diagnostics in hand, follow this remediation checklist to standardize on a single installer.

    Step 1: Remove every duplicate¶

    Uninstall Codex from each package manager you’ve ever used. Treat this like a clean slate.

    # npm install
npm uninstall -g @openai/codex

# Homebrew install (macOS/Linux)
brew uninstall codex

# winget install (Windows)
winget uninstall OpenAI.Codex

    Step 2: Reinstall from one source¶

    Pick one delivery channel—npm or Homebrew—and stick with it for future upgrades.

    # Recommended (cross-platform): npm
npm cache clean --force
npm install -g @openai/codex@latest

# Alternative (macOS/Linux): Homebrew
brew install codex

    Step 3: Verify in a fresh terminal¶

    Open a brand-new shell, then validate the binary path and version string. This clears shell caches automatically.

    # Confirm only one executable remains
which codex                  # macOS/Linux
Get-Command codex -All       # Windows PowerShell

# Check the reported version
codex --version

# Cross-check with npx
npx -y @openai/codex@latest --version

    What success looks like

    You should see a single path, and the codex --version output should match the npx result. If the CLI still prints an older number but npx reports the latest, you’ve hit the known version-reporting bug—functionally you’re already on the latest build.

    Common failure patterns and fixes¶

    SymptomLikely causeFix
    codex --version is old but npx is newVersion reporting bugTreat as cosmetic; track GitHub Issue #4986
    Fresh terminal still loads old versionPATH precedenceCheck echo $PATH; move the npm/global bin ahead of older locations
    Native Windows install keeps crashingExperimental supportReinstall inside WSL and run from the Linux layer
    VS Code or Cursor extension feels outdatedExtension version lagUpdate the extension manually and restart the editor
    How to update the VS Code extension
    1. Open the Extensions sidebar in VS Code
    2. Search for “OpenAI Codex”
    3. Click Update
    4. Fully restart VS Code (close all windows, then reopen)
    5. Sign back into Codex from the sidebar

    Key takeaways¶

    Codex CLI updates fail when duplicate binaries, shell caches, or a cosmetic version bug get in the way. Run the diagnostics, consolidate on one installer, and validate in a fresh terminal to end the loop.

    • Use which -a codex / Get-Command codex -All to locate duplicates
    • Purge all installs, then reinstall with a single package manager
    • Validate in a new shell and double-check with npx

    Future upgrades become straightforward once every machine sticks to the same installer (npm install -g @openai/codex@latest or brew upgrade codex).

    What to do next¶

    • Review the Codex CLI official docs for new capabilities
    • Watch the Codex GitHub repository for release notes and bug reports
    • If you also use the VS Code or Cursor extensions, keep them updated alongside the CLI
