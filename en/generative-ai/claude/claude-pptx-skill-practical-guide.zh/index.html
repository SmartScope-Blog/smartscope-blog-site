    Claude PPTX Skill 让你效率提升10倍！PowerPoint 自动化完全指南 [2025]¶

    • 每周花费 1-3 小时制作提案书、报告和内部资料的团队
    • 希望在保留公司模板的同时自动更新内容的用户

    阅读本文的收获（5分钟）¶

    1. 传统 AI 在演示文稿制作上失败的原因
    2. 使用 Claude PPTX Skill 实现工作流级别自动化的操作步骤

    与传统 AI 的决定性差异（What）¶

    传统 AI（ChatGPT 等）的常见失败¶

    • 各页幻灯片的设计风格不统一
    • 无法保留公司 PowerPoint 模板
    • 仅仅为了更新数字就需要从头重做

    根本原因：AI 无法直接操作 PPTX 结构，只是在表面上排列文本。

    Claude PPTX Skill 带来的变革¶

    Anthropic 的 Agent Skills / document-skills（2025年10月正式发布，可在 Claude.ai、Code 和 API 中使用）内置了 PowerPoint（pptx）技能，支持"演示文稿的创建和编辑"。

    • 保留模板的布局、配色和字体，仅更新内容
    • 从 Excel 数据自动生成/更新图表和表格
    • 从日语原稿生成多语言版本（布局保持不变）

    使用前需了解的关键限制（事实）¶

    坚持不夸大的事实描述。

    支持的套餐¶

    Claude.ai 的 Skills 功能（包括 PPTX 技能）在 Free / Pro / Max / Team / Enterprise 所有套餐中均可使用。前提条件是 Code execution and file creation 已启用。

    Enterprise / Team 套餐

    Organization Owner 需要在 Settings > Capabilities 中启用 Code execution，在 Organization settings > Skills 中启用技能。启用后，各成员可在 Customize > Skills 中切换各技能的开/关。

    与 Claude in PowerPoint 加载项的区别

    2026年2月发布的 Claude in PowerPoint（研究预览）是可以在 PowerPoint 应用中直接使用 Claude 的加载项，适用于 Pro / Max / Team / Enterprise 套餐（基于官方博客和 Microsoft AppSource 信息）。本文介绍的 Claude.ai PPTX 技能（在浏览器中生成/编辑 PPTX 的功能）是不同的产品。

    支持的文件格式¶

    PowerPoint：支持 .pptx 的创建和编辑（document-skills 的一部分）。

    文件大小限制（Claude.ai 浏览器版）¶

    单文件最大 30MB（上传/下载合计）。

    实际容量还取决于Token 限制（上下文长度），因此限制不是"多少页幻灯片"，而是"在 30MB 以内且在模型上下文范围内"。

    无幻灯片数量保证

    官方没有类似"100页以上也没问题"的幻灯片数量保证。本文使用的参考标准是"在30MB以内，一般商务资料的数十页到约100页是可行的"。

    Claude PPTX Skill 的使用方法（How）¶

    模式 0：初始设置（仅需一次）¶

    0-1. 启用文件创建功能¶

    1. 登录 Claude.ai（所有套餐均支持）
    2. 打开设置（右上角）
    3. 在 Feature Preview 或 Capabilities 部分，确认"文件创建和编辑（File Creation）"已启用

    内置的文档技能（pptx / docx / xlsx / pdf）在代码执行（File Creation）启用后默认开启。通常无需额外操作即可使用。如需禁用特定技能，可在 Customize > Skills 中单独切换开/关。

    Claude Code 用户

    在 Claude Code 中，技能通过在 .claude/skills/ 目录放置 SKILL.md 文件或从插件市场安装来使用。文档技能的源代码已在 anthropics/skills 公开，但与 Claude.ai 的内置技能使用方法不同。

    0-2. 附加文件的方法（Claude.ai）¶

    1. 打开新对话
    2. 点击消息输入区域的回形针图标（文件附件）
    3. 附加以下文件：
    4. 公司模板：company_template.pptx
    5. 现有资料：proposal_2024_Q3.pptx
    6. 数据文件：sales_2025_nov.xlsx 等

    手动流程（改善前）¶

    • 粘贴到 PPT 并调整：15分钟

    • 合计：每周30分钟 → 每年25-26小时

    Claude 工作流（改善后）¶

    步骤 1：附加模板和最新 Excel

    • weekly_report_template.pptx
    • sales_2025_w45.xlsx（最新一周的数据）

    步骤 2：向 Claude 发出指令（可直接复制粘贴）

    提示词示例

update the following with data from the attached `sales_2025_w45.xlsx`:
- Sales summary (Slide 2)
- Sales by product (Slide 3)
- Graph values (Slide 4)
Maintain the template's layout and design, only updating labels to this week's week number as needed.

    步骤 3：下载输出文件

    • Claude 生成更新后的 .pptx
    • 下载后抽查数字和图表，仅进行少量文字微调

    效果： 几分钟内完成草稿 → 每年节省 20 小时以上

    实际使用注意事项¶

    1. 最终输出必须经过人工审核¶

    数字、专有名词和法律用语需要人工核实。

    2. 注意文件大小和 Token 限制¶

    超过 30MB 的文件或包含大量备注的文件可能会被截断或概括。

    3. 设计模板时优先考虑"结构"¶

    对于周报模板，"哪些指标放在哪些幻灯片上"的规则越固定，替换效果就越稳定。

    4. 注意模板来源（安全性）¶

    Skills 是带有代码执行权限的功能。从外部收到的模板 PPTX 中可能嵌入了恶意指令（提示注入）。

    • 仅使用内部管理的可信模板
    • 从外部收到的 PPTX 在传递给 Claude 前先确认内容
    • 安装自定义技能时，提前检查 SKILL.md 和脚本的内容

    5. 注意更新可能导致行为变化¶

    Anthropic 可能会改进 Skills 和文件处理功能，如果作为内部标准使用，建议每年进行审查。

    常见问题（以操作方法为核心）¶

    Q1. 需要编程知识吗？

    不需要。使用浏览器版 Claude.ai，只需 (1) 确认文件创建功能已启用，(2) 附加 PPTX/Excel，(3) 用自然语言给出指令即可。

    Q2. 真的可以直接使用现有的公司模板吗？

    可以。附加 .pptx 模板并指示"使用此模板制作XX资料"。布局和配色通常会被保留，但无法100%保证，因此最终的设计调整需要人工完成。

    Q3. 实际可以处理多大规模的资料？

    限制因素是文件大小（30MB）和上下文限制，而不是幻灯片数量。对于一般商务资料，数十页到约100页是可行的，但建议在内部先用1-2个模式验证"运行稳定性"。

    Q4. 可以从 Claude Code / API 使用吗？

    可以。

    • API: 通过代码执行工具 + Skills API 的 container 参数实现 PPTX 生成和编辑。最新的 API 签名为 container={"skills": [{"type": "anthropic", "skill_id": "pptx", "version": "latest"}]}（参见官方 Skills API Quickstart）。
    • Claude Code: 在 .claude/skills/ 目录放置 SKILL.md 文件，或从插件市场安装。
