      CLAUDE.mdとは¶

      CLAUDE.mdは、Claude Codeがあなたのプロジェクトを理解するための指示ファイルです。 ファイルの内容はセッション開始時にUser Messageとして注入され、AIがプロジェクト固有のルールやワークフローを理解した上で支援を提供できます。

      仕組みについて

      CLAUDE.mdはSystem Promptではなく、会話の最初のメッセージとして扱われます。そのため「設定を書いたのに無視された」と感じる場合は、セッションが長くなるにつれてCLAUDE.mdの内容が古い文脈に押し流されている可能性があります。詳細なメカニズムは 注入メカニズム解説 を参照してください。

      重要: ファイル名は必ず CLAUDE.md（すべて大文字）にしてください。

      5分で始める最小構成¶

      Step 1: ファイルを作成する（30秒）¶

      プロジェクトのルートディレクトリで以下を実行：

      touch CLAUDE.md

      Step 2: 最小限の内容を追加（2分）¶

      以下の内容をCLAUDE.mdに記述：

      # プロジェクト設定

## 概要
このプロジェクトは[目的]のためのアプリケーションです。

## 技術スタック
- 言語: [Python/JavaScript/など]
- フレームワーク: [使用しているフレームワーク]

## 主要コマンド
- 開発サーバー起動: `npm run dev`
- テスト実行: `npm test`

## 注意事項
- [プロジェクト固有の重要な注意点]

      Step 3: 実際の例（2分）¶

      React プロジェクトの場合の具体例：

      # プロジェクト設定

## 概要
顧客管理システムのフロントエンドアプリケーション

## 技術スタック
- 言語: TypeScript
- フレームワーク: React 18
- スタイリング: Tailwind CSS

## 主要コマンド
- 開発サーバー起動: `npm run dev`
- テスト実行: `npm test`
- ビルド: `npm run build`

## 注意事項
- APIエンドポイントは環境変数で管理
- コンポーネントはsrc/components配下に配置

      Step 4: Gitに追加（30秒）¶

      git add CLAUDE.md
git commit -m "Add CLAUDE.md configuration"

      効果を実感する¶

      CLAUDE.mdを追加した後、Claude Codeに以下のような質問をしてみてください：

      Before（CLAUDE.mdなし）¶

      Q: 「新しいコンポーネントを追加して」
A: どのようなコンポーネントを、どこに追加すればよいでしょうか？

      After（CLAUDE.mdあり）¶

      Q: 「新しいコンポーネントを追加して」
A: src/components配下にTypeScriptとTailwind CSSを使用したコンポーネントを作成します。

      よくある質問¶

      Q: ファイル名は小文字でも動作しますか？¶

      A: 技術的には動作しますが、CLAUDE.md（大文字）が推奨されています。

      Q: どこに配置すべきですか？¶

      A: プロジェクトのルートディレクトリに配置してください。

      Q: 日本語で書いても大丈夫ですか？¶

      A: はい、日本語で記述できます。チームの言語に合わせてください。
