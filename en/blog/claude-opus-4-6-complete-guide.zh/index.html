    Beta版・有使用限制

    性能指标：

    • MRCR v2基准测试（8-needle、1M上下文）：76%（从Sonnet 4.5的18.5%大幅提升）
    • 长对话中的"上下文衰减"问题基本消除

    • 超过200K令牌后适用长文溢价（输入10/输出37.50）

    主要新功能¶

    多个Claude智能体并行分工协作的新功能。例如前端、后端、测试专员同时工作，可大幅提升复杂软件开发效率。

    Anthropic官方博客展示了Agent Teams自主构建C编译器并支持Linux内核的案例。原本需要人类数天的工作，通过多智能体并行处理缩短至数小时。

    可用环境：Claude Code（v2.1.32以上）、Cowork平台研究预览版。

    根据任务难度自动调节思考深度的新功能。从Extended Thinking进化而来，提供4个effort级别（low/medium/high/max）。默认为"high"。

    message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    effort="max",  # 应对最复杂的问题
    messages=[{"role": "user", "content": "复杂架构设计"}]
)

    成本优化：简单任务指定"low"可降低延迟和推理成本。

    长对话中自动摘要旧消息以压缩上下文的功能（Beta版）。理论上实现"无限对话"。开发者可控制压缩触发时机，大幅提升智能体应用的持续性。

    代码生成时的自我审查与调试功能增强，"首次运行即接近完成品"的频率大幅提高。SentinelOne的CLAO评价为"高级工程师级别的规划能力和策略适应性"。

    常见问题（FAQ）¶

    Q: 1M令牌上下文所有人都能用吗？

    A: 不是。1M令牌上下文为Beta版，需满足Tier要求。请查阅官方文档了解最新条件。

    Q: 超过200K令牌后计费如何变化？

    A: 上下文超过200K令牌时，整个请求切换为长文溢价（输入10、输出37.50每1M令牌）。即使"只超了一点"，全部令牌都按溢价计费。

    Q: 指定Data Residency（US-only）需要额外费用吗？

    A: 是的。指定inference_geo后会产生1.1倍（10%加价）的费用。1

    Q: Prompt Caching和Batch API能降低多少成本？

    A: Prompt Caching缓存读取为0.1x（约90%折扣），Batch API为50%折扣。两者组合使用可进一步大幅降低成本。2

    Q: 可以在GitHub Copilot/Bedrock/Vertex/Azure上使用吗？

    A: 可以。所有平台自2026年2月5日起即日可用：

    • GitHub Copilot
    • Amazon Bedrock
    • Google Vertex AI
    • Azure AI Foundry
