    GitHub Copilotデータ学習ポリシー変更（2026年4月24日施行）──オプトアウト手順と影響範囲¶

    GitHub Copilot 完全ガイド

    対象 / ポイント

    対象: GitHub Copilot Free / Pro / Pro+ を使用している開発者

    ポイント:

    • 2026年4月24日から、CopilotとのやりとりがAIモデル学習にデフォルトで使用される
    • Business / Enterprise / 学生プランは対象外
    • オプトアウトは github.com/settings/copilot から今すぐ設定可能
    • 4月24日施行 30日前通知のオプトアウト方式
    • プライベートリポも対象 Copilot使用中にやりとりしたコードが収集される
    • 1分で設定完了 Settings → Copilot → Privacy で無効化
    • 他ツールとの比較 Cursor・Claude Codeとの違い

    何が変わるのか¶

    2026年3月25日、GitHubはプライバシーポリシーとサービス利用規約の更新を発表しました1。4月24日から、Copilot Free / Pro / Pro+ ユーザーのインタラクションデータがAIモデルのトレーニングに使用されます。

    これはオプトイン（同意制）ではなくオプトアウト（拒否制）です。何もしなければ自動的に有効になります。

    影響を受けるプラン・受けないプラン¶

    プラン影響備考
    Freeありデフォルトで有効
    Pro（$10/月）ありデフォルトで有効
    Pro+（$39/月）ありデフォルトで有効
    Business（$19/ユーザー/月）なし契約条件で除外
    Enterprise（$39/ユーザー/月）なし契約条件で除外
    学生・教師（無料Pro）なし明示的に除外

    個人プラン（Free/Pro/Pro+）のユーザーが対象です。組織契約のBusiness/Enterpriseは従来通り除外されています。

    収集されるデータの範囲¶

    「インタラクションデータ」として以下が収集対象になります2：

    • Copilotに送信された入力（コードスニペット含む）
    • Copilotが生成した出力（受け入れ・修正したコード）
    • カーソル位置周辺のコードコンテキスト
    • ファイル名・リポジトリ構造・ナビゲーションパターン
    • Copilot機能との対話（チャット、インライン提案）
    • ユーザーフィードバック（評価）

    プライベートリポジトリも対象

    リポジトリ単体が「保存状態で」トレーニングに使われることはありません。しかしCopilot使用中にアクティブにやりとりしたプライベートリポジトリのコードスニペットは収集・使用される可能性があります。

    リポジトリ単位での制御はできません。オプトアウトはアカウント全体に対して行います。

    オプトアウト手順（1分で完了）¶

    設定ページ: github.com/settings/copilot

    手順¶

    1. GitHubにログイン
    2. github.com/settings/copilot にアクセス
    3. 「Privacy」 セクションまでスクロール
    4. トレーニング関連のトグル（「Allow GitHub to use my data for AI model training」 または類似の表記）をオフに切り替え

    以前「GitHub Copilot interaction data for product improvement」でオプトアウトしていた場合、その設定は引き継がれます。設定項目名はUIの更新で変わる場合があるため、「Privacy」セクション内のトグルを確認してください。

    確認ポイント

    設定変更後、ページをリロードしてトグルがオフのままか確認してください。組織のCopilot管理者設定とは独立した個人設定です。

    なぜオプトアウトが話題になっているのか¶

    GitHub Community Discussion #188488では232のダウンボートが付き、39件の投稿中、計画を支持したのはGitHub VP 1名のみでした3。

    コミュニティで挙がっている主な懸念は3つあります：

    • オプトアウト方式への批判: ユーザーの明示的同意なく有効化される設計。EUではGDPR上オプトイン（同意制）が一般的であり、法的リスクが指摘されている
    • 設定の発見しにくさ: 無効化設定へのアクセスが直感的でないという「ダークパターン」批判
    • リポジトリ単位の制御不可: プライベートリポの一部のみ除外する手段がない

    GitHubのMario Rodriguez CPO は「参加することで、開発ワークフローの理解度が向上し、より正確で安全なコードパターンの提案が可能になる」と説明しています1。

    他のAIコーディングツールとの比較¶

    ツールデータトレーニングポリシー
    GitHub Copilot (Free/Pro/Pro+)デフォルトでオプトイン。オプトアウト可
    CursorPrivacy Mode（デフォルトOFF）を有効にするとゼロデータ保持
    Claude Code (Anthropic)APIインプットはモデルトレーニングに使用しないと明記

    Cursorはprivacy modeの明示的なトグルがあり、Claude Code（Anthropic API）はそもそも入力データをトレーニングに使わないポリシーです。GitHubは自社の方針について、JetBrains・Microsoftなど他社の類似事例を引用しています1。

    まとめ¶

    4月24日の施行まで約1か月あります。オプトアウト設定は今すぐ確認できます。

    特に以下に該当する場合は早めの対応を推奨します：

    • 業務コードを個人アカウントのCopilotで扱っている場合: 機密コードがトレーニングデータに含まれるリスク
    • OSS以外のプライベートリポジトリで作業している場合: コードスニペットが収集対象
    • GDPR対象地域のユーザー: 正当な利益に基づく処理への異議申し立てが可能（GitHubのプライバシー窓口へ連絡）

    Business/Enterpriseプランへの移行も選択肢の一つですが、まずは設定ページでオプトアウトを確認するのが最も手軽です。
