      LLMコーディングベンチマーク比較【2026年4月版】¶

      GPT-5.4 / Claude / Gemini 3.1 をどう読むか¶

      この記事で学べること

      2026年4月16日時点で、どの比較軸がまだ有効で、どれが古くなったか GPT-5.4、Claude Opus 4.6 / Sonnet 4.6、Gemini 3.1 Pro の現在地 SWE-Bench Pro、Terminal-Bench 2.0、OSWorld、価格、長文脈の読み分け

      2026年2〜3月の比較記事は、もうそのままでは使いづらくなっています。最大の理由は、OpenAI側の主役が GPT-5.3-Codex から GPT-5.4 に切り替わったことです。加えて Anthropic では Opus 4.6 が旗艦、Sonnet 4.6 が日常運用の主力という二層構造がはっきりし、Google側は Gemini 3.1 Pro を complex tasks 向けの最上位モデルとして維持しています。

      したがって今やるべきなのは、「古いランキングを暗記すること」ではなく、どのベンチマークが今の製品選択に効くかを整理し直すことです。

      先に結論¶

      迷い方まずの答え
      CLI 自走・リポジトリ修正・長めの実装タスクGPT-5.4 / Codex 側が最も強い公開シグナル
      高性能な上位モデルで Claude を使うべきかOpus 4.6 は依然有力。ただし日常運用は Sonnet 4.6 も重要
      Google 系の長文脈・マルチモーダル・価格効率を見たいGemini 3.1 Pro はまだ有力。ただし Google の比較表は GPT-5.4 前
      古い GPT-5.3-Codex vs Opus 4.6 タイトルで十分か十分ではない。2026年4月時点では GPT-5.4 前提に直すべき

      まず前提を更新する¶

      OpenAI側¶

      OpenAIは 2026年3月5日に GPT-5.4 を発表し、ChatGPT・API・Codex に展開しています。公式には「GPT-5.3-Codex の frontier coding capabilities を取り込み、professional work 全体へ拡張した mainline reasoning model」と説明されています。

      つまり、今の OpenAI 側比較は GPT-5.4 を基準 に置くべきです。GPT-5.3-Codex は、もはや比較の中心ではなく GPT-5.4 の前世代の参照点 です。

      Anthropic側¶

      Anthropic は Claude Opus 4.6 を「most capable model to date」と位置づけています。長い multi-step tasks、agentic workflows、enterprise workflows が主戦場です。一方で Claude Sonnet 4.6 は、claude.ai と Claude Cowork の default model になっており、価格も $3 / $15 に据え置かれています。

      ここから分かるのは、Claude 側を1モデルだけで語ると実態を外すということです。

      • 旗艦比較では Opus 4.6
      • 実運用比較では Sonnet 4.6 も重要

      この二層で見る必要があります。

      Google側¶

      Google DeepMind の model card では、Gemini 3.1 Pro は「Google’s most advanced model for complex tasks」とされています。トークン文脈は 1M、出力は 64K。価格は Gemini API の paid tier で $2 / 12（<=200k tokens）**、**4 / $18（>200k） です。

      一方で重要なのは、Google の比較表は 2026年2月時点の表であり、OpenAI側は GPT-5.2 / GPT-5.3-Codex の列で止まっていることです。したがって、Gemini 3.1 の旧比較表をそのまま現在のリーダーボードとして読むのは危険です。

      今の比較で見るべき5軸¶

      1. SWE-Bench Pro¶

      リポジトリ課題解決に近い比較軸です。2026年4月時点で OpenAI は GPT-5.4 57.7% を公開し、比較対象として GPT-5.3-Codex 56.8%、GPT-5.2 55.6% を並べています。

      ここで重要なのは、OpenAI が 5.4 が 5.3-Codex を超えた と明確に示したことです。したがって、Codex系の評価記事を更新するなら、SWE-Bench Pro の主役はもう GPT-5.4 です。

      2. Terminal-Bench 2.0¶

      CLI / ターミナルでの自走能力です。OpenAI の公開値では GPT-5.4 75.1%。Anthropic の Opus 4.6 ページでは Terminal-Bench 2.0 65.4% が掲示されています。

      ここは現時点で最も素直に差が見える軸で、CLI 系は GPT-5.4 優位 と整理してよいです。

      3. OSWorld¶

      コンピュータ操作や GUI / ブラウザ系の比較軸です。旧来は「Claude が GUI に強い」という整理がされがちでしたが、OpenAI は GPT-5.4 で OSWorld-Verified 75.0% を公開し、Anthropic の Opus 4.6 は 72.7% を公開しています。

      つまり、GUI は Claude 優位 という古い結論は、そのままでは維持できません。

      4. 価格¶

      2026年4月時点の公開価格だけ並べると、上位モデルの価格感はこうなります。

      モデル入力出力補足
      GPT-5.4$2.50$15Codex / API / ChatGPT へ展開
      Claude Opus 4.6$5$251M context は Claude Platform beta
      Claude Sonnet 4.6$3$15Claude.ai / Cowork の default
      Gemini 3.1 Pro$2 or $4$12 or $18200k tokens を境に価格段階が変わる

      上位モデル比較では、GPT-5.4 は Opus 4.6 より安い。Claude 側でコストを抑えるなら、自然に Sonnet 4.6 が比較対象に入ります。Google 側は 短めのプロンプトではかなり競争力がある 価格です。

      5. 長文脈¶

      OpenAI の GPT-5.4 は 1.05M context window、ただし 272K を超える入力では課金倍率が変わる と API docs にあります。Anthropic は 1M token context window を beta として Opus 4.6 / Sonnet 4.6 の文脈を押しています。Gemini 3.1 Pro も 1M context を持ちます。

      したがって 2026年4月時点では、長文脈自体は差別化軸になりにくく、長文脈を何に使うか の方が重要です。

      • CLI実装・自走なら GPT-5.4
      • 長い enterprise workflow や丁寧な reasoning なら Opus 4.6
      • Google stack での multimodal / search / long context なら Gemini 3.1 Pro

      4モデルを一枚で置くとこう見える¶

      モデル今の位置づけ向きやすい場面
      GPT-5.4OpenAI の現行主力。GPT-5.3-Codex を吸収した比較基準CLI自走、コーディング、computer use、価格効率
      Claude Opus 4.6Anthropic の旗艦難しい multi-step task、上位 reasoning、enterprise workflows
      Claude Sonnet 4.6Claude.ai / Cowork の default日常開発、長セッション、コスパ重視の Claude 運用
      Gemini 3.1 ProGoogle の complex tasks 向け最上位Google系統合、multimodal、1M context、価格競争力

      2. 結論¶

      古い結論のうち、特に直すべきなのはここです。

      • ターミナルはCodex、GUIはClaude を固定結論として書かない
      • Claude を1モデル扱いしない
      • Gemini 3.1 の Google 比較表を「現時点の完全比較」と見なさない
