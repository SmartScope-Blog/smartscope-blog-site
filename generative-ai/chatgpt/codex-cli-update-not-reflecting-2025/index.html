    Codex CLI アップデートが反映されない？v0.99+対応 4手順で解決（キャッシュ/重複削除）¶

    Codex CLI 完全ガイド

    この記事で解決できること

    Codex CLI アップデートが反映されない原因の特定
    バージョン表示確認・シェルキャッシュクリア・重複削除・再インストールの4手順を実践
    今後同じ問題を起こさない予防策の理解

    npm install -g @openai/codex@latest や brew upgrade codex を実行したのに、codex --version が古いままで困っていませんか。実は Codex CLI アップデート がすぐに反映されないケースは珍しくありません。この記事では、重複インストール、パスキャッシュ、シェルのコマンドキャッシュ、バージョン表示バグなどの原因を診断し、キャッシュクリアと再インストール手順で4つの対処（バージョン確認→キャッシュクリア→重複削除→再インストール）で確実に最新版を使えるようにする手順をまとめます。

    この記事の対象者

    • npm や Homebrew で Codex CLI を更新したのに、古いバージョンが起動し続けている開発者
    • チーム内で Codex CLI のバージョン差異が頻発し、標準化に課題があるリードエンジニア

    この記事のポイント¶

    1. 実際に使用されている Codex 実行ファイルを特定できる
    2. 重複インストールやキャッシュ問題を解消できる
    3. 確実に最新版の Codex CLI を起動できる

    なぜ Codex CLI アップデートが反映されないのか¶

    Codex CLI のアップデートが反映されない理由は、主に以下の3つに集約されます。

    主な3つの原因¶

    npm、Homebrew、winget、手動バイナリなど複数の方法でインストールした結果、古いバージョンが PATH の前方に残り、そちらが優先されます。特に Windows では %AppData%\npm、C:\Program Files\Git\usr\bin、手動配置の実行ファイルが混在しやすい環境です。

    Bash や Zsh は一度実行したコマンドのパスをキャッシュします。アップデート後に新しいターミナルを開かないと古いパスが使われ続け、最新版が起動しません。この問題は hash -r（Bash）や rehash（Zsh）で解消できます。

    GitHub Issue #4986 で報告されているように、codex --version が誤った値を返す既知の不具合があります。実際には最新版がインストールされていても、表示だけが古い値になるケースです。

    Windows 環境の注意点

    Codex CLI の公式ドキュメントでは Windows 環境を実験的サポートと明記しています。ネイティブ Windows で不安定な場合は、WSL（Windows Subsystem for Linux）での運用が推奨されています。

    Codex CLIアップデートフローチャート（3分診断）¶

    codex cli update コマンドを実行したのに反映されないときのミニチェックリスト
    1. バージョン確認 — codex --version と npx -y @openai/codex@latest --version を比較し、表示バグかアップデート失敗かを切り分ける。
    2. 重複実行ファイルの洗い出し — which -a codex（macOS/Linux）または Get-Command codex -All（PowerShell）で複数パスが無いかを確認。重複があれば全てアンインストール。
    3. 再インストールとPATH確認 — npm install -g @openai/codex@latest または brew install codex 1本に統一し、新しいターミナルで which codex / Get-Command して1つだけになったことを確認。

    この3手順を最初に実施すれば、「codex cli update」系クエリで求められている即時の答えを提供できます。

    現状を診断する3つのコマンドセット¶

    まず、どの Codex 実行ファイルが実際に使われているかを特定しましょう。以下のコマンドで、システム内のすべての Codex 実行ファイルと npm によるインストール状況を確認します。

    # Windows PowerShell
Get-Command codex -All | Select-Object -Expand Source
where.exe codex
npm root -g
npm ls -g @openai/codex --depth=0

    # macOS / Linux
which -a codex
npm root -g
npm ls -g @openai/codex --depth=0

    Get-Command codex -All や which -a codex で複数のパスが表示された場合、重複インストールが原因です。/usr/local/bin/codex と ~/.npm-global/bin/codex が同時に存在する場合などは、PATH の順序に従って古いものが優先されます。

    npx を使った確認方法

    npx -y @openai/codex@latest --version を実行すると、PATH に依存せず npm registry の最新版を直接起動できます。このコマンドで新しいバージョンが表示されるのに、通常の codex --version が古い場合、PATH 問題が確定です。

    確実にアップデートする3ステップ¶

    診断結果をもとに、以下の手順で確実に最新版へ統一します。重複を解消し、単一の管理方法に統一することで、今後のアップデートもスムーズになります。

    Step 1: 重複を解消する¶

    まず、すべての Codex インストールをクリーンアップします。npm、Homebrew、手動配置など、すべての経路をチェックし、いったん削除します。

    # npm でインストールした場合
npm uninstall -g @openai/codex

# Homebrew でインストールした場合（macOS/Linux）
brew uninstall codex

# winget でインストールした場合（Windows）
winget uninstall OpenAI.Codex

    Step 2: 最新版をインストール¶

    重複を解消したら、1つの方法だけで最新版をインストールします。npm または Homebrew のどちらか一方に統一することを推奨します。

    # npm で統一する場合（推奨：クロスプラットフォーム）
npm cache clean --force
npm install -g @openai/codex@latest

# Homebrew で統一する場合（macOS/Linux）
brew install codex

    Step 3: 動作確認¶

    インストール後、新しいターミナルを開き、以下のコマンドで確認します。シェルのキャッシュをクリアするため、既存のターミナルではなく必ず新規に開いてください。

    # 実行ファイルのパスを確認（1つだけになっているはず）
which codex                  # macOS/Linux
Get-Command codex -All       # Windows PowerShell

# バージョンを確認（最新版が表示されるはず）
codex --version

# 念のため、npx で最新版と比較
npx -y @openai/codex@latest --version

    成功の確認ポイント

    which codex や Get-Command codex -All の結果が 1 つのパスのみになり、codex --version と npx の結果が一致すれば成功です。バージョン表示バグが存在するため、表示が古くても npx で最新版が確認できれば問題ありません。

    よくあるトラブルと対処法¶

    症状原因解決策
    codex --version が古いが npx は最新バージョン表示バグ実際は最新版が動作中。表示バグとして既知（Issue #4986）
    新しいターミナルでも古いままPATH の順序問題echo $PATH で npm のパスが前方にあるか確認。.bashrc や .zshrc で調整
    Windows で動作が不安定Windows は実験的サポートWSL 上でインストールし直す（公式推奨）
    VS Code/Cursor 拡張が古いまま拡張のバージョン不一致拡張を手動で更新し、VS Code を再起動。拡張側の設定も要確認
    VS Code 拡張の更新手順
    1. VS Code の拡張タブを開く
    2. OpenAI Codex 拡張を検索
    3. 「更新」ボタンをクリック
    4. VS Code を完全に再起動（ウィンドウを閉じて再度開く）
    5. Codex サイドバーでサインインし直す

    まとめ¶

    Codex CLI アップデートが反映されない問題は、重複インストール、シェルキャッシュ、バージョン表示バグのいずれか、または複合的な原因で発生します。診断コマンドで実行ファイルの所在を特定し、単一の管理方法に統一することが最短の解決策です。

    • which -a codex や Get-Command codex -All で重複を検出
    • すべてのインストールを削除後、npm または Homebrew のどちらか 1 つで再インストール
    • 新しいターミナルで動作確認し、npx と比較して最新版を確認

    今後のアップデートは、統一した方法（npm install -g @openai/codex@latest または brew upgrade codex）で実施すれば、同じ問題は起きません。

    次のステップ¶

    • Codex CLI 公式ドキュメントで最新機能を確認
    • Codex GitHub リポジトリでリリースノートや Issue をチェック
    • VS Code/Cursor 拡張も併用している場合は、拡張側のアップデートも忘れずに実施
