    AgentKitで変わるAIエージェント開発¶

    8分でエージェントを構築した実演¶

    OpenAI DevDay 2025で最も注目されたのが、エンジニアのクリスティーナ・ファンによる実演です。彼女はAgentKitを使い、わずか8分でDevDay来場者向けのQ&Aエージェントを構築しました。

    AgentKitの4つのコア機能¶

    ドラッグ&ドロップでワークフローを設計。条件分岐、ループ、データ変換をノードで配置できます。プレビュー実行、インライン評価設定、完全なバージョン管理に対応しています。

    埋め込み可能なチャットUIを提供。自社ブランドのデザインカスタマイズに対応し、数行のコードでWebサイトやアプリに統合できます。

    エージェントのパフォーマンスを測定。ステップごとのトレース記録、データセット評価、自動プロンプト最適化が可能です。サードパーティモデルの評価にも対応しています。

    Dropbox、Google Drive、Microsoft Teams、SharePoint などの人気プラットフォームとの統合を一元管理。管理ダッシュボードでセキュリティと権限を制御できます。

    実装の第一歩¶

    # AgentKit SDK基本例
from agents import Agent, Runner

agent = Agent(
    name="サポートエージェント",
    instructions="顧客の質問に日本語で回答する"
)

result = Runner.run_sync(
    agent,
    "製品Aの返品方法を教えて"
)
print(result.final_output)

    実践のコツ

    Agent Builderの事前構築テンプレート（カスタマーサポート、データ分析など）から開始すると、設計時間を80%削減できます。

    AgentKit連携例（クリックで展開）

    Ramp: 空白キャンバスから数時間で購買エージェントを構築。経費承認プロセスを自動化し、処理時間を90%削減しました。

    Box: 内部データ検索エージェントを実装。権限管理を維持しながら、1,000以上のドキュメントから情報を即座に抽出できるようになりました。

    HubSpot: AgentKitでBreeze AIツールを強化。顧客問い合わせへの応答品質が向上し、初回解決率が40%改善しました。

    ChatGPT内でアプリを直接実行する時代¶

    Apps SDKの革新性¶

    OpenAI DevDay 2025では、Model Context Protocol（MCP）に基づくApps SDKが発表されました。これにより、Spotify、Canva、Figma、Courser a、Booking.com、Zillow、ExpediaなどのアプリをChatGPT内で直接操作できます。

    実装の流れ

    1. MCPサーバーを作成してデータソースを接続
    2. Apps SDK設計ガイドラインに沿ってUIコンポーネントを構築
    3. Developer ModeでChatGPT内テスト実行
    4. 年内開始予定のアプリ審査へ提出

    参考情報

    DoorDash、Instacart、Uber、AllTrailsなどが対応予定。Apps SDKはOpenAI開発者ドキュメントで詳細確認できます（オープンソースで公開）。

    ユーザー体験の変化¶

    従来はアプリを個別起動していましたが、Apps SDK導入後は「Spotify、金曜のパーティー用プレイリスト作成」と指示するだけでアプリが自動起動。ChatGPTの文脈を理解した状態で処理を開始します。

    Apps SDKの技術的特徴¶

    • MCP準拠: オープンスタンダードに基づくため、他のプラットフォームでも動作可能
    • カスタムコード: 開発者が独自コードでインターフェースとチャットロジックを定義
    • バックエンド直接接続: データソースに安全に接続し、リアルタイムデータを取得

    新機能3点

    1. Slack統合¶

    チームチャネルから直接コード生成を依頼できます。@Codexとタグ付けするだけで、会話の文脈を理解し、適切な環境を選択して作業を完了します。

    # Slackでの使用例
@Codex この関数のユニットテストを追加して

    2. Codex SDK¶

    自社ワークフローへの組み込みが可能になりました。わずか数行のコードでGPT-5-Codexの能力を統合できます。

    // Codex SDK使用例（TypeScript）
import { CodexClient } from '@openai/codex-sdk';

const client = new CodexClient();
const result = await client.generate({
  prompt: 'React Hookで状態管理を実装',
  language: 'typescript',
  context: '既存のコンポーネント構造を考慮'
});

    3. 管理ダッシュボード¶

    使用量分析、権限制御、環境管理、コードレビュー品質の追跡が可能になりました。Business、Edu、Enterpriseプランで利用できます。
