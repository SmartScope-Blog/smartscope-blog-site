      Claude Code 完全ガイド

      Claude Code Windowsインストール: WSL不要（3分設定）¶

      概要¶

      Claude Codeが 2025年にネイティブWindows版 をリリースしました！これまでWSL（Windows Subsystem for Linux）が必須でしたが、Windowsで直接動作するようになり、Windowsユーザーの利便性が大幅に向上しました。最新バージョンは claude --version で確認できます。

      ネイティブWindows版のメリット

      • WSL不要 - Windowsで直接動作
      • ワンコマンドインストール - 公式インストーラーがすべて処理（Node.jsの個別インストール不要）
      • Git Bash必須 - Claude CodeはGit Bashを使ってネイティブ動作（未インストール時は自動案内）
      • Windows Package Manager (winget) 対応 - winget upgrade Anthropic.ClaudeCode で簡単アップデート
      • 安定性の継続的改善 - .bashrcファイル処理、コンソールウィンドウのフラッシング、OAuthトークン有効期限、プロキシ設定、bashサンドボックスエラー、日本語IMEサポート等の修正

      インストール手順¶

      1. PowerShellで公式インストーラーを実行¶

      管理者権限は不要です。PowerShellで以下のコマンドを実行：

      irm https://claude.ai/install.ps1 | iex

      これはAnthropicの公式インストーラーです。Claude Codeのバイナリが ~/.local/bin に自動配置されます — Node.jsの個別インストールは不要です。

      代替方法：Windows CMD または WinGet でインストール **Windows CMD**（PowerShellよりコマンドプロンプトを使いたい場合）： 
      curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
      **WinGet**（自動更新されません — 定期的に `winget upgrade` を実行）： 
      winget install Anthropic.ClaudeCode

      npm installは非推奨

      以前の npm install -g @anthropic-ai/claude-code によるインストール方法は、公式ドキュメントでdeprecated（非推奨）と明記されています。上記の公式インストーラーを使用してください。

      2. 認証¶

      インストール後、claude を初回実行するとブラウザが自動で開き、OAuth認証が行われます。APIキーの管理は不要です。

      claude

      Git Bashが必要

      Windows版Claude CodeはGit Bashを使ってネイティブ動作します。Gitが未インストールの場合、インストーラーがインストールを案内します。Git for WindowsにはGit Bashが含まれています。

      3. インストール完了後の重要な手順¶

      必須：ターミナルを再起動

      インストール完了後、必ずターミナルウィンドウを閉じて新しく開き直してください。PATHの変更を反映させるために必要です。

      claude doctor でインストールを検証

      インストール後、claude doctor を実行すると、インストールの種類・バージョン・一般的な設定問題を検出できます。

      トラブルシューティング¶

      claudeコマンドが認識されない場合¶

      症状¶

      PS C:\Users\username> claude
claude : 用語 'claude' は、コマンドレット、関数、スクリプト ファイル、
または操作可能なプログラムの名前として認識されません。

      または：

      Native installation exists but C:\Users\<ユーザー名>\.local\bin is not in your PATH.

      原因と解決方法（公式インストーラーの場合）¶

      公式インストーラーはバイナリを ~/.local/bin に配置します。このパスがPATHに含まれていないとコマンドが見つかりません。

      1. .local\bin がPATHに含まれているか確認

        $env:PATH -split ";" | Where-Object {$_ -like "*.local*"}

      2. 含まれていなければPATHに追加

        # .local\bin をユーザーPATHに追加
[Environment]::SetEnvironmentVariable("PATH", "$env:PATH;$env:USERPROFILE\.local\bin", [EnvironmentVariableTarget]::User)

# 現在のセッションにも反映
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"

      3. 動作確認

        claude --version

      それでも動作しない場合¶

      直接実行でテスト： 

      & "$env:USERPROFILE\.local\bin\claude.exe"

      旧npmインストールのトラブルシューティング 以前 npm (`npm install -g @anthropic-ai/claude-code`) でインストールした場合、バイナリの場所が異なります： 1. **npmグローバルプレフィックスを確認** 
      npm config get prefix
# 出力例: C:\Users\username\npm-global
      2. **npmのPATHを確認・修正** 
      $env:PATH -split ";" | Where-Object {$_ -like "*npm*"}

# npm-globalがPATHに含まれていない場合は追加
[Environment]::SetEnvironmentVariable("PATH", "$env:PATH;C:\Users\<ユーザー名>\npm-global", [EnvironmentVariableTarget]::User)
      Note: npmインストールは非推奨です。公式インストーラーへの移行を推奨します。

      クリップボード画像貼り付けについて¶

      現状の制限事項¶

      Windows+Shift+S の制限

      Windows+Shift+Sでキャプチャした画像をCtrl+Vで貼り付ける機能は、多くのユーザーから動作しないと報告されています。ファイル経由での画像共有を推奨します。

      代替方法¶

      1. Ctrl+VではなくALT+Vを使う
      2. Claude Codeでは ALT+V でクリップボードから画像を貼り付け可能
      3. Ctrl+V はテキストのみの貼り付け — バグではなく仕様

      4. スクリーンショットをコピーしてALT+Vを試してください

      5. ファイルとして保存してドラッグ&ドロップ（最も確実）

      6. スクリーンショットをファイルに保存

      7. Claude Codeのウィンドウに直接ドロップ

      8. ファイルパスを指定

        画像を見てください: C:\Users\username\Pictures\screenshot.png

      検証結果¶

      Ctrl+Vでは画像を貼り付けできません

      最新のネイティブWindows版でも、Windows+Shift+Sでキャプチャした画像のCtrl+V貼り付けは動作しません — Ctrl+Vはテキストのみです。 画像貼り付けには ALT+V を使用するか、ファイル経由で共有してください。

      将来の展望¶

      ネイティブWindows版のリリースにより、今後のアップデートでクリップボード機能の改善が期待されます。元のクリップボード貼り付けIssue（#1361）は部分的に対応済みで、一部プラットフォームではALT+Vが動作しますが、WindowsでのCtrl+V画像貼り付けは引き続き要望中です（#26679、#22068）。

      WSL版との比較¶

      ネイティブWindows版のメリット¶

      • 簡単セットアップ - WSLのインストール・設定が不要
      • Windows統合 - PowerShell/コマンドプロンプトで直接動作
      • 軽量 - Linux環境を立ち上げる必要なし
      • Windows権限 - Windowsのファイルシステムに直接アクセス

      WSL版のメリット¶

      • 既存Linux環境活用 - 既にWSL環境がある場合は追加設定不要
      • Bashツールのサンドボックス - WSL 2はサンドボックスによるセキュリティ強化に対応（WSL 1は非対応）
      • パッケージ管理 - apt/yumなどLinuxパッケージマネージャーが使用可能
      • コンテナ連携 - Dockerなどとのシームレスな統合

      公式の推奨¶

      Anthropic公式：ネイティブインストールが推奨

      公式ドキュメントでは「Native Install」が推奨インストール方法として記載されています。Windows環境では、ネイティブGit BashとWSLの両方がサポートされています。WSL 2はサンドボックスによるセキュリティ強化が利点、ネイティブGit Bashはシンプルなセットアップが利点です。

      使い分けの指針¶

      こんな時はネイティブWindows版（公式推奨）

      • 一般的な用途（公式推奨のため）
      • Windows開発をしている
      • Git Bashが使える環境
      • シンプルに使いたい

      こんな時はWSL版

      • 既にWSL環境が構築済み
      • Dockerと連携したい
      • Linuxツールチェーンが必要

      まとめ¶

      Claude CodeのネイティブWindows版により、WSLの設定なしで簡単に利用開始できるようになりました。クリップボード画像貼り付けには制限がありますが、ファイル経由での画像共有は問題なく動作します。

      用途に応じてネイティブ版とWSL版を使い分けることで、より効率的な開発環境を構築できます。

      チェックリスト

      • PowerShellで公式インストーラー実行（irm https://claude.ai/install.ps1 | iex）
      • claude を実行してブラウザで認証
      • ターミナルを再起動
      • PATHの設定確認（.local\bin がPATHに含まれているか）
      • claudeコマンドの動作確認
