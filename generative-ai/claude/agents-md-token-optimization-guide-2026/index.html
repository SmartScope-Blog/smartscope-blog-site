    Step 1: メインAGENTS.md設計¶

    # AGENTS.md（最適化版）

## 🎯 利用可能スキル

| スキル | トリガー例 | 用途 |
|--------|-----------|------|
| `article-workflow` | 「記事作成」「執筆」 | **記事作成オーケストレーター** |
| `coding-standards` | 「規約」「PEP 8」 | コーディング規約 |
| `dev-workflow` | 「ビルド」「コミット」 | 開発ワークフロー |
| `fact-checker` | 「ファクトチェック」 | 情報検証 |

## ⚡ オンデマンドロード方式

**重要**: スキルは要求時にのみロードされます。トリガーキーワードで自動起動。

## 📋 コア制約（絶対厳守）

1. **事実確認最優先** - 虚偽情報完全排除
2. **手動更新日時禁止** - git-revision-date-pluginに統一
3. **YAML Escaping** - `{{ }}` 必須
4. **生成物整理** - 専用ディレクトリに集約

## 🗣️ コミュニケーションポリシー

- 内部処理・思考・実装時の注釈：英語可
- 最終提案・報告・要約・ユーザー説明：日本語提供

# Article Workflow Skill

## トリガー条件
「記事作成」「記事を書いて」「新規記事」「執筆して」

## 使用場面
- 新規記事作成リクエスト
- 記事改訂作業
- コンテンツワークフロー調整

## 4段階ワークフロー

### Phase 1: リサーチ・準備
- 対象読者設定
- コア情報収集
- 構成案作成

### Phase 1.5: 構成レビュー（読者体験チェック）
- ペルソナ検証
- 情報階層確認

### Phase 2: 執筆
- 基本構造制約遵守
- SEO最適化

### Phase 2.5: 品質チェック
- ファクトチェック完了
- 最終UXチェック

## ⚠️ クリティカルゲート
- **Phase 1 ゲート**: ペルソナ・コアメッセージ確定
- **Phase 1.5 ゲート**: 読者体験レビュー完了
- **Phase 2 ゲート**: 基本構造制約遵守
- **Phase 2.5 ゲート**: ファクトチェック完了

## 他スキル連携
- `fact-checker`: 情報検証時
- `seo-analyzer`: SEO分析時
- `reader-experience-reviewer`: UX評価時

    トークン効率化の実装パターン¶

    パターン1: コンテキスト境界の明確化¶

    ## 📋 コア制約（絶対厳守）
1. **事実確認最優先** - 虚偽・誇張情報の完全排除
2. **クリティカルシンキング** - バイアス除去と論理整合性
3. **手動更新日時禁止** - git-revision-date-pluginに統一
4. **YAML Escaping** - GitHub Actions変数は `{{ }}` 必須

    パターン2: 優先度ベースタスク管理¶

    ## ⚡ AI/Agent駆動開発コンテキスト

- ❌ 従来の時間見積もり（Week、Day等）は使用禁止
- ✅ 優先度と依存関係で管理（P0=最優先、P1=高、P2=中、P3=低）
- ✅ タスクは数時間〜1日で完了可能性高

    パターン3: 参照分離アーキテクチャ¶

    ## 📌 詳細情報参照先

詳細ルール・ガイドラインはスキルに移管。該当トリガーで自動起動：

| 旧ファイル | 移管先スキル |
|-----------|------------|
| `development-workflow.md` | `dev-workflow` |
| `article-guidelines.md` | `article-proofreader` |
| `coding-style.md` | `coding-standards` |
