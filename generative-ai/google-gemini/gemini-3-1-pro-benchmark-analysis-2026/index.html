      Google Gemini 完全ガイド

      Gemini 3.1 Proは今どう見るべきか【2026年4月版】¶

      Googleの公開表を GPT-5.4 時代に読み替える¶

      この記事でやること

      Gemini 3.1 Pro が 2026年4月時点でも有力なのかを整理 Google の 2026年2月比較表を、そのまま現在の順位表として読まないための注意点を示す GPT-5.4 と Claude Opus 4.6 / Sonnet 4.6 が出揃った後の位置づけを整理する

      2026年2月に出た Gemini 3.1 Pro「16項目中13で首位」 の話は、いまそのままでは読みにくくなっています。理由は単純で、Googleの比較表が作られた時点では OpenAI 側が GPT-5.2 / GPT-5.3-Codex の列で止まっており、GPT-5.4 がまだ出ていなかった からです。

      だからといって Gemini 3.1 Pro が無価値になったわけではありません。今やるべきなのは、launch-day の勝敗表を持ち続けることではなく、Gemini 3.1 Pro の強みが今も有効な領域と、比較表として古くなった部分を分けて読むことです。

      先に結論¶

      まず知りたいこと今の答え
      Gemini 3.1 Pro はまだ Google の上位モデルかはい。Google は今も most advanced model for complex tasks と位置づけている
      Gemini 3.1 の旧比較表を今の OpenAI / Claude 比較として使えるかそのままでは使えない。OpenAI 側が GPT-5.4 前だから
      Gemini 3.1 Pro の魅力は何か1M context、multimodal、Google系の agentic / search / coding 文脈、価格競争力
      いま broad comparison に入れる価値があるかある。ただし「2026年2月の勝敗表」ではなく「2026年4月の選択肢」として扱うべき

      まず、Google公式が今も言っていること¶

      Google DeepMind の model card では、Gemini 3.1 Pro は 「Google’s most advanced model for complex tasks」 とされています。入力は text / audio / image / video、文脈は 1M、出力は 64K。つまり、Google側の上位ポジションは今も維持されています。

      価格も Gemini API では依然競争力があります。

      条件入力出力
      <= 200k tokens$2.00$12.00
      > 200k tokens$4.00$18.00

      この価格は、OpenAI の GPT-5.4（$2.50 / 15）や Anthropic の Opus 4.6（5 / $25）と比較しても、十分に比較対象に入ります。

      ただし、比較表は「2026年2月の表」である¶

      Googleの eval methodology には、はっきりと次の前提が書かれています。

      • 非Geminiモデルの結果は、原則として providers' self reported numbers
      • Claude Opus 4.6、Sonnet 4.6、GPT-5.2 については maximum thinking / reasoning settings available を採用
      • そして当然ながら、その文書自体が February 2026 のもの

      ここで重要なのは、OpenAI 側の列が GPT-5.4 ではない ことです。つまり、Gemini 3.1 Pro の表は「古い表だから無意味」ではないが、今の OpenAI / Claude / Google の三者比較を確定させる表ではない ということです。

      2026年4月時点で変わったこと¶

      OpenAI側は GPT-5.4 に更新された¶

      OpenAI は GPT-5.4 を ChatGPT・API・Codex にロールアウトし、SWE-Bench Pro、Terminal-Bench 2.0、OSWorld-Verified を含む新しい公開表を出しています。

      たとえば OpenAI の公式公開値では、

      • SWE-Bench Pro (Public): 57.7%
      • Terminal-Bench 2.0: 75.1%
      • OSWorld-Verified: 75.0%

      となっており、これは Gemini 3.1 Pro を論じるときの前提も変えます。旧記事で GPT-5.3-Codex 不在や未公開を中心に組み立てていた論点は、そのままでは弱くなっています。

      Claude側は Opus 4.6 と Sonnet 4.6 の二層整理が必要になった¶

      Anthropic は Opus 4.6 を最上位モデルとして示しつつ、Sonnet 4.6 を claude.ai と Claude Cowork の default model にしています。つまり 2026年4月時点では、Claude 側は Opusだけでなく Sonnet も実務比較に入れるべき です。

      Gemini 3.1 Pro を比較するなら、相手は単なる Claude ではなく、

      • 上位比較では Opus 4.6
      • 実運用比較では Sonnet 4.6

      という二層になります。

      それでも Gemini 3.1 Pro が有力な理由¶

      1. Google 自身が complex tasks の最上位に据えている¶

      これはまだ変わっていません。大規模な multimodal 情報、長文脈、戦略的 planning を含む difficult tasks を担うモデルとして、Google は Gemini 3.1 Pro を前面に出し続けています。

      2. 1M context と multimodal の組み合わせ¶

      Gemini 3.1 Pro の魅力は、単に「安い」ことではありません。1M context + multimodal + Google系ツール連携との相性 をまとめて持っている点が、他社比較でも残ります。

      3. 価格競争力¶

      上位モデル帯の中では、Gemini 3.1 Pro の価格はまだ十分に魅力があります。特に <=200k tokens では $2 / $12 なので、OpenAI の GPT-5.4 より入力も出力も少し軽いです。

      逆に、今は過信しない方がいい点¶

      1. 「16項目中13で首位」を現在の総合順位と見なすこと¶

      これはやりすぎです。Google の比較表は 2026年2月の時点表 であり、今の OpenAI 側旗艦は GPT-5.4 です。したがって、launch-day のシェア用メッセージを今の総合順位に変換するのは危険です。

      2. GPT-5.3-Codex を今の OpenAI 比較対象とすること¶

      もう中心ではありません。今の OpenAI 側比較は GPT-5.4 を前提に組み直すべきです。

      3. Claude を1モデルとして扱うこと¶

      今の Claude 側は、Opus 4.6 と Sonnet 4.6 を分けて見ないと解像度が落ちます。

      2026年4月時点の使い分け¶

      使い方まず有力な候補
      Google系統合、multimodal、1M context を活かしたいGemini 3.1 Pro
      CLI 自走と coding の公開シグナルを重視するGPT-5.4
      難しい multi-step enterprise task を丁寧に回したいClaude Opus 4.6
      Claude をコストも見ながら日常運用したいClaude Sonnet 4.6

      まとめ¶

      • Gemini 3.1 Pro は、2026年4月時点でも Google側の有力上位モデル です
      • ただし Google の比較表は GPT-5.4 前 なので、そのまま現在の順位表としては使えません
      • 今の Gemini 記事は、13/16勝った という launch-day の物語より、今どの用途で有力か に寄せた方が長持ちします
      • 2026年4月時点の broad comparison では、GPT-5.4 / Opus 4.6 / Sonnet 4.6 / Gemini 3.1 Pro の4点で読むのが自然です
